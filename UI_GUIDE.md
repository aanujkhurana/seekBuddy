# UI Guide

## Design Direction

Modern, minimal, premium, and trustworthy.

The app should feel like a clean productivity tool, not a hacky bot.

## Visual Style

- Dark mode first.
- Light mode later.
- Rounded cards.
- Clear spacing.
- Soft borders.
- Calm accent colour.
- No clutter.
- No aggressive growth-hack copy.

## Core Screens

### 1. Welcome

Purpose:
Explain what the app does and what it does not do.

Copy:
```txt
Apply faster, stay in control.
Use AI to draft better applications and review everything before submitting.
```

### 2. SEEK Connection

Elements:
- Open SEEK login button
- Session status
- Explanation that login happens directly in browser

### 3. Job Preferences

Fields:
- Keywords
- Location
- Remote/hybrid/on-site
- Salary minimum
- Max applications per run
- Excluded keywords
- Review before submit

### 4. Resume

Elements:
- Resume file picker
- Resume preview/extracted text later
- Optional base cover letter

### 5. AI Settings

Modes:
- Built-in AI
- Bring your own key

For normal users:
```txt
Built-in AI
Simple setup. Limited daily usage.
```

For advanced users:
```txt
Bring your own API key
Your key is stored only on this computer.
```

### 6. Run Screen

Elements:
- Start button
- Stop button
- Current status
- Job currently being processed
- Logs
- Progress count

### 7. Review Screen

Elements:
- Job title
- Company
- Match score
- Red flags
- Generated cover letter
- Screening answers
- Submit / Skip / Edit

### 8. History

Elements:
- Applied jobs
- Date applied
- Company
- Role
- Link
- Notes
- Export button later

## Tone

Use direct, calm language.

Good:
```txt
Waiting for you to review this application.
```

Bad:
```txt
Crushing applications with AI automation!
```
