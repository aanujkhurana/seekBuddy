# /build Command

Use this command to build a feature.

## Required Input

- Feature name
- User problem
- Acceptance criteria

## Process

1. Read relevant docs.
2. Create feature branch.
3. Inspect current implementation.
4. Implement smallest useful version.
5. Add user-friendly errors.
6. Add logs where useful.
7. Run checks.
8. Prepare PR summary.

## Rules

- Do not mix unrelated refactors.
- Do not hardcode secrets.
- Do not reduce safety defaults.
- Do not make automation hidden.
- Do not remove review-before-submit default.
